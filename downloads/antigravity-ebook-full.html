Mengenal Interface Antigravity

Sebelum mulai bekerja, ada baiknya kita kenali dulu tampilan Antigravity. Ini akan membantu Anda lebih nyaman saat nanti mulai "tek-tok" dengan AI.

Dua Panel Utama

Begitu aplikasi terbuka, Anda akan melihat layar terbagi menjadi dua bagian besar:

Tampilan 2-Panel Antigravity

Panel Kiri: Workspace (Area Kerja)

Ini tempat di mana kode dan file project Anda ditampilkan. Ada dua tab biasanya:

  • Code: Menampilkan source code (HTML, CSS, JavaScript, dll)
  • Preview: Menampilkan hasil aplikasi yang sedang dibuat

Fokus Anda di panel kiri ini adalah untuk MELIHAT HASIL, bukan edit manual. Biarkan AI yang ngurusin code-nya.

Panel Kanan: Chat (Obrolan dengan AI)

Ini adalah tempat Anda mengetik perintah. Interface-nya mirip WhatsApp:

  • Ada kolom kecil di bawah untuk menulis prompt
  • AI akan merespons di atas kolom tersebut
  • Conversation history tersimpan, bisa scroll ke atas untuk lihat percakapan sebelumnya

Workflow: Accept, Reject, atau Always Proceed?

Saat AI selesai memproses request Anda dan menghasilkan code, biasanya akan muncul notifikasi yang menanyakan persetujuan.

Accept Button Workflow

Anda akan melihat 2 pilihan utama:

  • "Accept" (biru) - Klik ini kalau Anda setuju dengan perubahan yang diusulkan AI
  • "Reject all" - Klik ini kalau Anda tidak setuju dan ingin batalkan perubahan

    • Tip Praktis:

    Untuk project pembelajaran atau eksperimen, biasanya langsung klik Accept saja. AI jarang ngasih code yang merusak. Kalau ada bug, nanti tinggal minta AI buat fix lagi.

    Opsi "Always Proceed" untuk Workflow Lebih Cepat

    Kalau Anda merasa bolak-balik klik Accept itu memperlambat workflow, ada solusinya:

    Always Proceed Option

    Pada notification box, ada opsi "Always Proceed" - ketika Anda enable ini, AI akan otomatis apply semua perubahan tanpa konfirmasi lagi.

    Kapan pakai Always Proceed?

    • Ketika Anda sedang iterasi cepat (testing ide, coba-coba fitur)
    • Ketika Anda percaya diri dengan prompt yang Anda berikan
    • Ketika Anda tidak khawatir dengan perubahan yang destructive (karena bisa di-undo)

    Kapan JANGAN pakai Always Proceed?

    • Ketika project sudah production/dipakai orang lain
    • Ketika request Anda kompleks dan butuh review manual
    • Ketika Anda masih belajar dan ingin tahu apa yang AI ubah

    Tips Pro 2: Pilih Mode "Planning" (Biar Gak Bolak-Balik)

    Di sebelah kiri kolom chat, ada menu kecil bertuliskan Mode.

    Planning vs Fast Mode

    Ada dua pilihan:

  • Fast (Cepat): AI langsung "hajar" nulis kode. Kalau salah, ya salah.
  • Planning (Perencanaan): AI akan mikir dulu, bikin rencana, dan nanya ke kamu: "Bos, saya mau bikin gini, setuju gak?"

    • Saran Saya:

    Gunakan selalu PLANNING MODE.

    Kenapa? Karena AI akan menampilkan preview rencananya dulu sebelum buang-buang "tenaga" (kuota) buat nulis kode.

    Kalau rencananya salah, kamu bisa koreksi di awal.

    Ini jauh lebih hemat limit daripada pakai mode Fast -> Salah -> Ulangi -> Habis Limit.

    Kamus Istilah (File-File Aneh)

    Selama proses pembuatan, kamu mungkin akan melihat file-file aneh bermunculan di foldermu. Nggak usah panik, ini artinya:

    1. File Laporan (Dokumen Proyek)

    Ini adalah file komunikasi antara kamu ("Bos") dan AI ("Mandor").

    • task.md: Ini Daftar Tugas (To-Do List). Isinya checklist apa saja yang harus dikerjakan AI.
    • implementation_plan.md: Ini Proposal Rencana. Sebelum kerja, AI biasanya bikin rencana dulu di sini. Kalau kamu setuju, baru dia kerjakan.
    • walkthrough.md: Ini Laporan Pertanggungjawaban. Setelah selesai kerja, AI akan nulis di sini: "Bos, saya sudah ngerjain A, B, C. Hasilnya begini (ada screenshot)."

    2. File Bangunan (Kodingan)

    Ini adalah bahan bangunan aplikasimu.

    • index.html: Ini Bangunan Utama. Kalau mau buka aplikasi, cari file ini.
    • style.css: Ini Cat & Dekorasi. Warna, font, tata letak, semuanya diatur di sini.
    • app.js / script.js: Ini Listrik & Otak. Yang bikin tombol bisa diklik, hitungan bisa jalan, itu file ini.
    • widget / data.js: Kadang ada file tambahan buat data atau fitur tempelan.

    Kesimpulan:

    Kalau kamu bingung, cukup cari index.html. Itu adalah pintunya. Sisanya? Biar AI yang urus.

    Bab 3.5: Melihat Proses Sebenarnya - Brainstorming sampai Debugging

    (Dokumentasi Lengkap: Tek-Tok dengan AI)

    Di bab sebelumnya saya sudah menjelaskan teori tentang bagaimana berkomunikasi dengan AI. Sekarang saya ingin menunjukkan seperti apa proses sebenarnya ketika membuat aplikasi - dari awal sampai selesai.

    Ini adalah dokumentasi nyata ketika saya membuat aplikasi "Dr. Anti Bokek" - sebuah personal finance tracker untuk tracking keuangan harian. Semua screenshot di sini adalah capture asli dari percakapan saya dengan Antigravity.

    Apa itu Dr. Anti Bokek?

    Dr. Anti Bokek adalah aplikasi web sederhana untuk membantu saya mengelola keuangan personal sehari-hari.

    Masalah yang ingin diselesaikan:

    • Saya punya budget bulanan yang sudah terstruktur (fixed expenses, wants, savings)
    • Tapi saya kesulitan tracking pengeluaran harian - apakah saya masih "on track" atau sudah "tekor"
    • Kadang di akhir bulan baru sadar ternyata sudah overspending

    Ini adalah "Senjata Rahasia" Anda. Tidak perlu hapal coding, cukup copy-paste mantra ini untuk membuat AI bekerja 10x lebih efektif.

    1. Mantra "Arsitek" (Untuk Memulai Project)

    Gunakan saat: Baru mau bikin aplikasi dari nol.

    "Halo, saya ingin membuat aplikasi berbasis web untuk [Tujuan Aplikasi, misal: kasir warung].

    Tolong berikan saya struktur folder dan file apa saja yang dibutuhkan.

    Gunakan HTML, CSS, dan JS vanilla saja agar mudah. Buat desainnya modern dan responsif untuk HP."

    2. Mantra "Tukang Dekor" (Untuk Mempercantik Tampilan)

    Gunakan saat: Tampilan aplikasi masih kaku seperti tahun 1990.

    "Tolong ubah desain file CSS-nya. Saya ingin gayanya 'Modern & Clean'.

    Gunakan font 'Inter' atau 'Poppins' dari Google Fonts.

    Berikan efek bayangan (box-shadow) lembut pada kartu dan tombol.

    Pastikan tombolnya enak dipencet di layar HP (padding besar)."

    3. Mantra "Satpam Logic" (Untuk Mencegah Error)

    Gunakan saat: Ada hitungan matematika yang salah atau error aneh.

    "Coba cek lagi logika hitungan di fungsi hitungTotal.

    Pastikan kamu [Jelaskan aturan, misal: tidak boleh ada angka negatif].

    Buatkan saya fungsi 'Guardian' untuk memfilter input yang aneh-aneh.

    Jika ada error, tolong jelaskan kenapa, dan berikan kode perbaikannya secara lengkap (jangan potong-potong)."

    4. Mantra "Detektif Error" (Saat Mentok)

    Gunakan saat: Aplikasi tidak jalan, layar putih, atau console merah semua.

    "Saya dapat error ini di Console: [Paste Error Disini].

    Tolong analisa penyebabnya. Jelaskan kepada saya seperti menjelaskan ke anak umur 5 tahun.

    Lalu perbaiki kodenya. Saya tidak mahu tahu teknisnya, pokoknya harus jalan."

    5. Mantra "Deploy" (Saat Mau Pamer)

    Gunakan saat: Project sudah jadi dan mau di-online-kan.

    "Project saya sudah selesai. Semua file ada di folder ini.

    Pastikan file utamanya bernama index.html.

    Apakah ada file gambar atau script yang path-nya salah (broken link)? Tolong cek dan perbaiki sebelum saya upload ke Netlify."

    Tips Tambahan: "The Golden Rule"

    Jangan pernah bilang: "Buatin aplikasi dong." (Terlalu umum, AI bingung).

    Tapi bilanglah: "Buatin tombol warna merah yang kalau diklik dia akan memunculkan alert 'Halo'." (Spesifik, AI senang).

    Jadilah Bos yang Spesifik. Anak buah (AI) akan bekerja sempurna.
